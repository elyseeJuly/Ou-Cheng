### 阶段 2：意象词库数据引擎重建

#### 2.1 [NEW] `scripts/build-imagery.js`
- **目的**: 从 `public/data/poetry.json`（33,000 首）中自动提取高频意象词组，生成权威的意象词频数据和 **共现关联矩阵**。
- **核心算法**:
  1. 对每首诗词正文进行分词（基于预设的 ~500 个古典意象种子词表）。
  2. 统计每个意象词的全局出现频次。
  3. 构建 **共现矩阵**: 对于每首诗中同时出现的意象词对 `(A, B)`，计数 `cooccurrence[A][B]++`，最终筛选出高频共现对。
- **输出产物**: 覆写 `public/data/imagery.json`，新增 `relatedWords` 字段：
  ```json
  {
    "word": "明月",
    "frequency": 1823,
    "relatedWords": [
      { "word": "相思", "strength": 0.82 },
      { "word": "清酒", "strength": 0.65 },
      { "word": "离愁", "strength": 0.58 }
    ]
  }
  ```

#### 2.2 [MODIFY] [types.ts](file:///Users/quantumrose/Documents/Emberois/ou-cheng/types.ts)
- 在 `ImageryItem` 接口中新增 `relatedWords?: { word: string; strength: number }[]` 字段。

### 阶段 3：词云意象连线交互实现

#### 3.1 [MODIFY] [WordCloud3D.tsx](file:///Users/quantumrose/Documents/Emberois/ou-cheng/components/library/WordCloud3D.tsx)
- **联想连线渲染**: 当用户点击某个意象词后：
  1. 从 `imageryItems` 中查找该词的 `relatedWords` 数组。
  2. 在 Three.js 场景中，使用 `THREE.Line` 或 `THREE.BufferGeometry` 绘制从选中词到各关联词的**弧线连线**，线宽和透明度与 `strength` 值正相关。
  3. 选中词放大并变为高亮色，关联词等比例突出显示，非关联词减淡（降低透明度至 0.15）。
  4. 连线颜色梯度：使用金色（`#d4af37`）到朱砂红（`#b22222`）的渐变。
- **交互细节**:
  - 点击关联词可切换焦点（新词成为中心，展开新的关联网络）。
  - 再次点击当前选中词或点击空白区域，清除所有连线并恢复默认旋转球体视图。
- **性能保护**: 每次连线最多展示 8 个最高 `strength` 的关联词，避免线条过度密集。

### 阶段 4：一键导入文稿功能

#### 4.1 [NEW] `components/creator/ImportModal.tsx`
- **智能导入弹窗组件**，提供两种导入方式：
  1. **粘贴导入**: 用户将诗词文本粘贴到大文本框中。
  2. **文件导入**: 支持拖放或选择 `.txt` 文件。
- **智能识别引擎**: 分析导入文本的行数、每行字数，自动推断诗词类型：
  - 4 行 × 5 字/行 → `jueju_5` (五言绝句)
  - 4 行 × 7 字/行 → `jueju_7` (七言绝句)
  - 8 行 × 5 字/行 → `lvshi_5` (五言律诗)
  - 8 行 × 7 字/行 → `lvshi_7` (七言律诗)
  - 14 行 → `sonnet` (十四行诗)
  - 其他长短句 → 尝试匹配 `cipai.json` 中的词牌字数 → 若匹配则为 `cipai`，否则为 `free`。
- **预览与确认**: 识别结果实时展示在弹窗下方（如"已识别为：七言绝句"），用户可手动修正后确认导入。
- **导入动作**: 调用 `savePoem()` 存入 localStorage，并自动跳转至作品库。

## 🚫 五、 AI 执行约束与红线

> [!IMPORTANT]
> 1. **外科手术式修改**: 修复 Bug 时仅触碰必须修改的代码行，严禁顺手重构相邻无关逻辑。
> 2. **词云性能红线**: 意象连线渲染必须在 16ms 以内完成（保持 60fps），连线数量硬限 ≤ 8 条。
> 3. **数据不可退化**: `build-imagery.js` 脚本生成的新 `imagery.json` 必须为 `imagery.json` 的超集。
> 4. **导入容错**: `ImportModal` 必须优雅处理空文本、乱码、超长文本等边界条件，禁止崩溃。
